# Canada Visa Automation - Production Design

## Overview
Multi-client visa appointment monitoring system with web admin panel and client-facing portal. Deploy to Render.com with Resend for email and Telegram for notifications.

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                        Render.com                               │
│  ┌─────────────┐    ┌────────────────────────────────────────┐ │
│  │  Flask App │───▶│  VisaAutomation Workers (threads)       │ │
│  │  (waitress)│    │  - Each client runs in daemon thread   │ │
│  └──────┬──────┘    │  - Screenshot capture per client      │ │
│         │           └────────────────────────────────────────┘ │
│  ┌──────┴──────┐                                               │
│  │   Routes    │                                               │
│  │ /login      │                                               │
│  │ /           │───▶ Admin UI (multi_user.html)               │
│  │ /generate_* │                                               │
│  │ /client/*   │───▶ Client UI (client_form.html)            │
│  └─────────────┘                                               │
└─────────────────────────────────────────────────────────────────┘
         │
    ┌────┴────┬──────────────┐
    ▼         ▼              ▼
┌────────┐ ┌────────┐   ┌──────────┐

└────────┘ └────────┘   └──────────┘
```

## Components

### 1. Flask Application (`app.py`)
- **Auth**: Session-based with ADMIN_PASSWORD
- **Admin Routes**: generate link, approve/reject, start/stop agents, view status
- **Client Routes**: submit request, poll status, view screenshot
- **State**: In-memory `automation_instances` + `client_tokens` dicts

### 2. VisaAutomation (`main.py`)
- Playwright Chromium (headless)
- Per-client thread with screenshot folder
- Properties: `is_running`, `current_action`, `action_log`, `new_date`, `current_date`

### 3. Notifications
- **Resend API**: Email notifications when earlier date found
- **Telegram Bot**: Push notifications to client chat IDs

### 4. Templates
- `multi_user.html` - Admin dashboard (terminal style)
- `client_form.html` - Client submit + monitor view
- `login.html` - Admin auth

## Data Flow

### Client Submission
1. Client visits `/client/<token>` (generated by admin)
2. Fills form → POST `/client_submit`
3. State → "pending" in `client_tokens`
4. Admin sees in "PENDING_REQUESTS" panel
5. Admin clicks "Approve" → automation starts in thread

### Status Polling
1. Client polls `/client_status/{token}` every 4s
2. Returns: `is_running`, `current_action`, `action_log`, `new_appointment`
3. If `new_appointment` found → trigger notification

### Notifications Trigger
```
Date found → 
  if notification_email: send_resend_email()
  if telegram_configured: send_telegram()
```

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `SECRET_KEY` | Yes | Flask session secret |
| `ADMIN_PASSWORD` | Yes | Admin login |
| `RESEND_API_KEY` | Yes | Resend API key |
| `TELEGRAM_BOT_TOKEN` | No | Telegram bot token |
| `FLASK_DEBUG` | No | "false" for prod |
| `PORT` | No | 8080 default |

## Error Handling

- **Automation crash**: Log error, set `is_running=False`, notify admin
- **Network failure**: Retry 3x with exponential backoff, then sleep 5min
- **Soft ban**: Sleep 10min, reset poll count
- **Resend failure**: Log, continue (don't block automation)
- **Telegram failure**: Log, continue

## Testing Strategy

1. **Import test**: `python -c "from main import VisaAutomation"`
2. **App starts**: `python app.py` (check no errors)
3. **Auth works**: Login with correct/incorrect password
4. **Client flow**: Generate link → submit → approve → check status

## Deployment (Render)

### render.yaml
```yaml
services:
  - type: web
    name: visa-ctrl
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: waitress-serve --port=$PORT --host=0.0.0.0 app:app
    envVars:
      - key: FLASK_DEBUG
        value: false
      - key: PORT
        value: 8080
```

### Requirements.txt additions
```
resend>=1.0.0
```
